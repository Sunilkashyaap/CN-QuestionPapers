Q2-A · Scenario: IPv4 Fragmentation + TTL + Routing 5 M | CO3
🌐 Packet's Journey Across the Internet
A computer in Mumbai sends a large IP datagram of 4000 bytes to a server in Delhi. The packet passes through three routers. The first router connects to an Ethernet network with MTU 1500 bytes. Each router decrements the TTL. The initial TTL is set to 5.
  1. What happens when the 4000-byte datagram reaches the first router with MTU 1500 bytes? Explain fragmentation with fragment sizes.
  2. What is the purpose of the TTL field and what happens when it reaches zero?
  3. Which protocol does the router use to send an error back to the source when TTL expires?
  4. At which device is the fragmented datagram reassembled?
  5. If the packet uses OSPF for routing, how does the router determine the best path?

✓ Model Answers

  1. Fragmentation: Since MTU = 1500 bytes and IP header = 20 bytes, each fragment can carry 1480 bytes of data. A 4000-byte datagram (with 20-byte header = 3980 bytes data) is split into 3 fragments: Fragment 1 = 1500 bytes (offset 0, MF=1), Fragment 2 = 1500 bytes (offset 185, MF=1), Fragment 3 = 1040 bytes (offset 370, MF=0). All fragments share the same Identifier field.
  2. TTL: TTL (Time To Live) is decremented by 1 at each router. It prevents packets from circulating forever in routing loops. When TTL = 0, the router discards the packet.
  3. When TTL expires, the router sends an ICMP Time Exceeded (Type 11) message back to the source IP address.
  4. Reassembly occurs only at the final destination host, not at intermediate routers. The destination buffers all fragments and uses the Identifier + Fragment Offset + MF flag to reconstruct the original datagram.
  5. OSPF routers use the Dijkstra's Shortest Path First (SPF) algorithm. Each router has a complete link-state database of the topology. It builds a shortest-path tree with itself as root and installs the best routes into the routing table. Metric = cost (based on bandwidth).
Q2-B · Scenario: UDP vs TCP + Sockets 5 M | CO4
🎮 Online Gaming Server
An online multiplayer game server needs to send real-time position updates to 500 players every 50ms. At the same time, it handles in-game purchases that must be 100% reliable. The game developer must decide which transport protocol to use for each use case.
  1. Which protocol — TCP or UDP — should be used for real-time position updates? Justify with reasons.
  2. Which protocol should handle in-game purchases? Why?
  3. Explain how a UDP socket sends data (steps in the process).
  4. What is the purpose of a port number in this context?
  5. Which UDP header field detects corruption in position update packets?

✓ Model Answers

  1. UDP for position updates. Reasons: (1) Real-time — even a 50ms retransmission delay is unacceptable in fast games. (2) A slightly outdated position is better than a delayed one. (3) UDP has no connection overhead, lower latency. (4) The next update will correct any missed position anyway.
  2. TCP for purchases. Reasons: (1) Money transactions must arrive reliably — no lost packets acceptable. (2) TCP guarantees in-order, error-free delivery via sequence numbers and ACKs. (3) TCP's reliability mechanisms (retransmission, checksums) ensure transaction integrity.
  3. UDP socket steps: (1) Create socket: socket(AF_INET, SOCK_DGRAM). (2) Fill destination address and port. (3) Send using sendto() — no connection needed. (4) Wait for reply with recvfrom(). (5) Close socket. No handshake, no connection state.
  4. A port number identifies which specific application/process on the host should receive the datagram. E.g., game server listens on port 27015, purchase service on port 443. Without ports, the OS wouldn't know which running application to deliver the packet to.
  5. The Checksum field (16 bits) in the UDP header detects corruption. It is computed over the header + data. If the checksum at receiver doesn't match, the packet is discarded.

Q2-B · Scenario: TCP Congestion + Timers 5 M | CO4
📡 File Transfer During Network Congestion
A user downloads a large file over TCP. Initially the network is fast, but midway through, the network becomes congested — several packets are lost. TCP detects this via a retransmission timeout and adjusts its behavior accordingly.
  1. Describe TCP's Slow Start phase. How does CWND grow?
  2. What happens to CWND and ssthresh when a timeout is detected?
  3. Which TCP timer detects that a packet has been lost and triggers retransmission?
  4. Explain the difference between Leaky Bucket and Token Bucket in controlling the download traffic.
  5. After timeout, TCP enters Slow Start again. When does it switch to Congestion Avoidance?

✓ Model Answers

  1. Slow Start: TCP starts with CWND = 1 MSS. For each ACK received, CWND increases by 1 MSS → so CWND doubles each RTT (exponential: 1→2→4→8...). This continues until CWND reaches ssthresh (slow start threshold).
  2. On Timeout: ssthresh is set to half of current CWND. CWND is reset to 1 MSS. TCP re-enters Slow Start from scratch.
  3. The Retransmission Timer (also called Timeout Timer). It starts when a segment is sent. If no ACK arrives before it expires, the segment is assumed lost and retransmitted. The timer value is dynamic, based on RTT estimates.
  4. Leaky Bucket: Outputs traffic at a constant, fixed rate regardless of input — smooths all bursts. Good for strict rate enforcement. Token Bucket: Tokens accumulate at a fixed rate; packets consume tokens. Allows bursts up to the bucket size. More flexible — useful when occasional fast bursts are acceptable but average rate is controlled.
  5. After Slow Start, TCP switches to Congestion Avoidance when CWND ≥ ssthresh. In Congestion Avoidance, CWND grows linearly by 1 MSS per RTT (instead of doubling), carefully probing for more bandwidth without causing another congestion event.

Q2-A · Scenario: RIP vs OSPF + Routing Table 5 M | CO3
🗺️ ISP Choosing a Routing Protocol
A large ISP has a network with 200 routers spread across India. Their current protocol RIP is causing slow convergence and routing loops during link failures. The network team is considering switching to OSPF. Some routers are Cisco-only branches where EIGRP could also be used.
  1. Why is RIP unsuitable for a 200-router network? Give two specific reasons.
  2. How does OSPF achieve faster convergence than RIP? Explain the role of LSA flooding.
  3. What is the maximum hop count limitation of RIP and why is it a problem here?
  4. For Cisco-only branches, why might EIGRP be preferred over OSPF?
  5. What metric does OSPF use to choose the best path, and how is it calculated?

✓ Model Answers

  1. RIP limitations for 200 routers: (1) Max hop count = 15 — any destination more than 15 hops away is unreachable, which is very likely in a 200-router network. (2) Slow convergence — RIP updates every 30 seconds; when a link fails, it takes minutes for all routers to learn the new topology (count-to-infinity problem can cause loops).
  2. OSPF convergence: When a link changes, the affected router immediately floods an LSA (Link State Advertisement) to ALL other routers. Every router updates its topology database and reruns Dijkstra's SPF algorithm. This happens in seconds, not minutes. Unlike RIP, OSPF doesn't wait for a 30-second timer.
  3. RIP's max hop count is 15. Hop count of 16 = infinity (unreachable). In a 200-router network spanning India, many paths will exceed 15 hops — those destinations will show as unreachable in RIP, which is unacceptable.
  4. EIGRP advantages for Cisco: (1) Faster convergence using DUAL algorithm with pre-computed backup (Feasible Successor) routes. (2) Uses incremental updates — only sends changes, not full table. (3) Supports unequal-cost load balancing. However, EIGRP is Cisco-proprietary so it only works with Cisco hardware.
  5. OSPF uses cost as its metric, calculated as: Cost = Reference Bandwidth / Interface Bandwidth (typically 100 Mbps / actual bandwidth). Lower cost = better path. A 1 Gbps link has cost 1; a 10 Mbps link has cost 10. OSPF always prefers high-bandwidth paths.

You just turned on your laptop and opened a browser to visit www.google.com. Your laptop has no IP address yet. Trace the complete sequence of network events that happen — from getting an IP address to receiving the webpage.

🌐 Host Sending Data Outside Its Network
A newly connected laptop in an office network wants to send data to a web server located outside its network. The laptop has no IP address yet. The office uses OSPF for routing between routers, and the packet must cross 3 routers to reach the server. The packet size is 3000 bytes but the first link has MTU = 1500 bytes.
  1. How does the laptop get its IP address? Name the protocol and describe it in 4 steps.
  2. What role does ARP play before the laptop sends the first packet?
  3. The 3000-byte packet must cross an MTU-1500 link. What happens? How many fragments are created?
  4. What happens if TTL becomes zero at a router before the packet reaches its destination?
  5. Why is OSPF preferred over RIP for this multi-router network?

✓ Model Answers

  1. DHCP — DORA process: (1) Discover: Laptop broadcasts from 0.0.0.0 to 255.255.255.255 requesting an IP. (2) Offer: DHCP server responds with an available IP address, subnet mask, gateway, DNS. (3) Request: Laptop broadcasts to confirm it accepts the offered IP. (4) Acknowledge: Server sends DHCPACK confirming the assignment. Laptop now has a valid IP.
  2. ARP's role: The web server is outside the laptop's network, so the packet must go to the default gateway (router). ARP is used to find the router's MAC address. Laptop broadcasts "Who has IP 192.168.0.1?" (router's IP). Router replies with its MAC. Laptop then sends the IP packet wrapped in an Ethernet frame addressed to the router's MAC.
  3. Fragmentation: Header = 20 bytes, so payload per fragment = 1500 − 20 = 1480 bytes. Data = 3000 − 20 = 2980 bytes. Fragment 1: 1480 bytes data (offset=0, MF=1). Fragment 2: 1480 bytes data (offset=185, MF=1). Fragment 3: 20 bytes data (offset=370, MF=0). 3 fragments total. All share same Identifier field.
  4. If TTL = 0, the router drops the packet and sends an ICMP Time Exceeded message (Type 11) back to the laptop's IP address. The message reports which router discarded it. The laptop knows the packet didn't reach its destination.
  5. OSPF vs RIP: (1) OSPF has no hop count limit; RIP max = 15 hops. (2) OSPF converges in seconds via immediate LSA flooding; RIP takes minutes with 30-second update cycles. (3) OSPF uses bandwidth-based cost; RIP only counts hops — a slow 56kbps link costs the same as 1Gbps. (4) OSPF is more scalable for enterprise networks.

🏦 Online Banking with TCP
An online banking application uses TCP to ensure secure and reliable money transfers. During peak hours, some packets are delayed and occasionally lost. However, the system still ensures that every transaction completes correctly and in the right order.
  1. Why is TCP preferred over UDP for banking transactions? Give at least 3 reasons.
  2. Explain the role of sequence numbers and acknowledgments in ensuring reliable delivery.
  3. How does TCP handle a lost packet? Describe the mechanism step by step.
  4. Which TCP flags are used during connection establishment? Name and explain each.
  5. Name any two TCP-based applications besides banking that require reliability.

✓ Model Answers

  1. TCP preferred for banking: (1) Reliability: TCP guarantees every byte arrives via ACKs and retransmission — a lost ₹10,000 transfer packet cannot just disappear. (2) Ordering: TCP sequence numbers ensure transactions arrive in the exact order sent — important for debit then credit operations. (3) Error checking: Checksum on every segment. (4) Flow + Congestion control: Prevents overloading the server. UDP has none of these.
  2. Sequence Numbers + ACKs: Sender assigns a sequence number to each byte. The receiver's ACK number tells the sender "I've received up to byte X, send me X+1 next." If ACK for byte X doesn't arrive within the retransmission timeout, the sender retransmits from byte X. This ensures no data is silently lost.
  3. Handling packet loss: (1) Sender transmits segment with sequence number X. (2) Retransmission timer starts. (3) If no ACK arrives before timeout → packet is assumed lost. (4) Sender retransmits the same segment. (5) Alternatively, if receiver sends 3 duplicate ACKs for byte X → fast retransmit (sender retransmits X immediately without waiting for timeout). (6) ssthresh = CWND/2, CWND resets, congestion control adjusts.
  4. Connection establishment flags: (1) SYN (Synchronize) — Client sends to initiate connection and share its initial sequence number. (2) SYN-ACK — Server responds, acknowledging client's SYN and sending its own ISN. (3) ACK (Acknowledgment) — Client confirms receipt of server's SYN. Connection is now established. The FIN flag is used for termination, not establishment.
  5. Two TCP-based applications: (1) HTTPS/HTTP (web browsing) — web pages must load completely and in order. (2) FTP (File Transfer Protocol) — files must arrive completely without corruption. Others: SMTP (email), SSH (secure shell), database queries.
