Propositional Logic
प्रस्तावात्मक तर्क (Propositional Logic)
🔥 Very High ▼
English
Propositional Logic (also called Boolean Logic or Statement Logic) is the simplest form of knowledge representation. It deals with propositions — statements that can be either TRUE or FALSE.
Key Components:
▸ Proposition: A declarative sentence that is either true or false. E.g., "It is raining" is a proposition. "Where are you?" is NOT.
▸ Atomic Proposition: Simple, single-fact statement. E.g., P = "Sun is hot"
▸ Compound Proposition: Built from atomic ones using connectives.
Logical Connectives (Truth Table):
P Q ¬P P ∧ Q P ∨ Q P → Q P ↔ Q
T T F T T T T
T F F F T F F
F T T F T T F
F F T F F T T
Inference Rule — Modus Ponens:
If P → Q is true, and P is true → Then Q must be true
TA-2 Exam Example
Let: H = High temperature, A = Power Available, F = Fan On, C = Room Cools
Rules: H ∧ A → F (fan on if temp high and power available)
        F → C (if fan on, room cools)
Given: H = T, A = T
Step 1: H ∧ A = T → Apply rule 1 → F = T
Step 2: F = T → Apply rule 2 → C = T
Conclusion: Room cools down. ✓
Limitations: Cannot express "All", "Some", "None" — for those, use First Order Logic.
हिंदी
प्रस्तावात्मक तर्क (Propositional Logic) ज्ञान प्रतिनिधित्व का सबसे सरल तरीका है। इसमें ऐसे वाक्यों (propositions) का उपयोग होता है जो सिर्फ TRUE या FALSE हो सकते हैं।
मुख्य बातें:
▸Proposition: एक ऐसा वाक्य जो सत्य या असत्य हो। जैसे: "बारिश हो रही है" — यह proposition है।
▸Connectives: ¬ (NOT), ∧ (AND), ∨ (OR), → (Implication), ↔ (Biconditional)
▸Modus Ponens: यदि P → Q सत्य है और P सत्य है, तो Q भी सत्य होगा।
परीक्षा उदाहरण
मान लो: H = तापमान ज़्यादा है, A = बिजली उपलब्ध है, F = पंखा चालू है, C = कमरा ठंडा होता है
नियम 1: H ∧ A → F (अगर तापमान ज़्यादा और बिजली है तो पंखा चलेगा)
नियम 2: F → C (पंखा चले तो कमरा ठंडा होगा)
दिया है: H = T, A = T → F = T → C = T
निष्कर्ष: कमरा ठंडा होता है। ✓
सीमाएँ: "सभी", "कुछ" जैसे quantifiers को express नहीं कर सकता — इसके लिए FOL चाहिए।

Forward & Backward Chaining
अग्र और पश्च तर्कसाधन (Forward & Backward Chaining)
🔥 Very High ▼
English
Forward Chaining (Data-Driven Reasoning):
Starts with known facts and applies inference rules repeatedly to derive new facts, until the goal is reached. It moves from facts → conclusion.
1
Start with all known facts in the knowledge base.
2
Find all rules whose IF part (conditions) match the current facts.
3
Fire those rules — add the THEN part as new facts.
4
Repeat until goal is found or no more rules can fire.
Example — Loan Approval
Facts: Alice has stable job, high credit score, no pending loans
Rule 1: Stable Job → Good Income → New Fact: Good Income
Rule 2: Good Income + High Credit → Financially Stable → New Fact: Financially Stable
Rule 3: Financially Stable + No Loans → Eligible for Loan → Goal Reached ✓
Backward Chaining (Goal-Driven Reasoning):
Starts with the goal and works backwards to find supporting facts. It moves from goal → supporting facts.
1
Set a goal/hypothesis to prove.
2
Find rules whose THEN part matches the goal.
3
The IF part of those rules becomes new sub-goals.
4
Repeat until all sub-goals are matched with known facts.
Example — Medical Diagnosis
Goal: Prove "patient has flu"
Rule: Fever + Cough → Flu → Sub-goals: Does patient have fever? Does patient have cough?
Check facts: Fever ✓, Cough ✓ → Flu confirmed ✓
Comparison Table:
Feature Forward Chaining Backward Chaining
Approach Data-driven Goal-driven
Starts with Known facts Hypothesis (goal)
Moves towards Conclusion Supporting facts
Best for Large datasets with many rules Specific problem-solving
Example Medical diagnosis from symptoms Crime investigation
हिंदी
Forward Chaining (आगे का तर्क):
यह ज्ञात तथ्यों से शुरू होता है और नियम लागू करते हुए नए तथ्य निकालता है जब तक लक्ष्य न मिल जाए। यह data-driven approach है।
उदाहरण: Alice के पास stable job है → Good Income → Financially Stable → Loan Eligible ✓

Backward Chaining (पीछे का तर्क):
यह लक्ष्य से शुरू होकर पीछे की ओर जाता है और देखता है कि लक्ष्य को सिद्ध करने के लिए कौन से तथ्य चाहिए। यह goal-driven approach है।
उदाहरण: Goal = "मरीज़ को flu है" → Rule: Fever + Cough → Flu → Facts check करो → Confirmed ✓

अंतर याद रखो:
Forward = Facts → Conclusion (Data-Driven)
Backward = Goal → Facts (Goal-Driven)

Bayes' Theorem (with Numericals)
बेयज़ प्रमेय (Bayes' Theorem)
🔥 Very High ▼
English
Bayes' Theorem calculates the probability of an event A given that event B has already occurred. It is the foundation of probabilistic reasoning in AI.
Formula:
P(A | B) = [ P(B | A) × P(A) ] / P(B)
Components:
▸P(A) — Prior probability: probability of A before seeing evidence B
▸P(B|A) — Likelihood: probability of seeing B if A is true
▸P(B) — Evidence: total probability of B happening
▸P(A|B) — Posterior: updated probability of A after seeing B
P(B) = P(B|A) × P(A) + P(B|¬A) × P(¬A)
Solved Numerical — TA-2 Q2c Pattern
Given:
P(Fire) = 0.002, P(¬Fire) = 0.998
P(SmokeAlarm | Fire) = 0.85, P(SmokeAlarm | ¬Fire) = 0.1
P(Sprinklers | Fire) = 0.6, P(Sprinklers | ¬Fire) = 0.02

Find: P(Fire | SmokeAlarm ∧ Sprinklers)

Step 1: Use Bayes with two independent pieces of evidence:
Numerator = P(SA|Fire) × P(Sp|Fire) × P(Fire)
            = 0.85 × 0.6 × 0.002 = 0.00102

Denominator = Numerator + P(SA|¬Fire) × P(Sp|¬Fire) × P(¬Fire)
                = 0.00102 + (0.1 × 0.02 × 0.998)
                = 0.00102 + 0.001996 = 0.003016

P(Fire | SA ∧ Sp) = 0.00102 / 0.003016 ≈ 0.338 ≈ 33.8%
⚡ Exam Tip
For two-evidence Bayes problems: Numerator = product of all likelihoods × prior. Denominator = same for Fire + same for ¬Fire. Then divide.
हिंदी
Bayes' Theorem एक mathematical formula है जो हमें बताता है कि जब कोई evidence (B) देखी जाए, तो किसी घटना (A) की probability कितनी होगी।
सूत्र:
P(A | B) = [ P(B | A) × P(A) ] / P(B)
प्रत्येक term का अर्थ:
• P(A) = Prior — B देखने से पहले A की probability
• P(B|A) = Likelihood — यदि A सत्य है तो B कितनी संभावना से होगा
• P(A|B) = Posterior — B देखने के बाद A की नई probability

याद रखने का तरीका:
Posterior = (Likelihood × Prior) / Evidence

परीक्षा में हमेशा step-by-step लिखो: पहले numerator निकालो, फिर denominator, फिर divide करो।

Bayesian Networks
बेयज़ियन नेटवर्क (Bayesian Networks)
⚡ High ▼
English
A Bayesian Network (BN) is a probabilistic graphical model that represents a set of variables and their conditional dependencies using a Directed Acyclic Graph (DAG).
3 Key Components (FAI-3 Q5B asks exactly this):
▸Nodes (Variables): Each node represents a random variable (e.g., disease, symptom, weather)
▸Edges (Directed Links): An arrow from node A to node B means A causally influences B (A is a parent of B)
▸Conditional Probability Tables (CPT): Each node stores the probability of being in each state given its parent nodes
Classic Example — Lung Cancer
Smoking → Lung Cancer ← Pollution
Lung Cancer → Cough

P(Smoking=Yes) = 0.3, P(Pollution=High) = 0.4
P(Lung Cancer | Smoking=Yes, Pollution=High) = 0.80
P(Cough | Lung Cancer=Yes) = 0.90

The network lets us compute: Given person coughs, what is probability they have lung cancer?
Applications: Medical diagnosis, spam filtering, fraud detection, weather prediction, self-driving cars.
हिंदी
Bayesian Network एक graphical model है जो variables और उनके बीच की conditional dependencies को DAG (Directed Acyclic Graph) के रूप में दर्शाता है।
3 मुख्य भाग:
1. Nodes: Random variables को represent करते हैं (जैसे: बीमारी, लक्षण)
2. Edges: Causal relationship दर्शाते हैं (तीर की दिशा में causation)
3. CPT (Conditional Probability Table): हर node की probability उसके parent nodes के आधार पर

उदाहरण: Smoking → Lung Cancer → Cough
अगर कोई खाँसता है, तो Lung Cancer की probability निकाली जा सकती है।

उपयोग: Medical diagnosis, Spam detection, Weather prediction

Unification & Lifting
एकीकरण और उन्नयन (Unification & Lifting)
⚡ High ▼
English
Unification:
Unification is the process of finding a substitution (a mapping of variables to values) that makes two or more logical expressions identical.
Rules of Unification:
▸Constants unify only if identical: a = a ✓, a = b ✗
▸A variable can unify with any constant: x = a gives {x → a}
▸Functions unify if their arguments unify: F(x,a) = F(b,y) gives {x→b, y→a}
▸Occurs Check: x cannot unify with F(x) — circular
Expression 1 Expression 2 Substitution
P(x, y) P(a, b) {x → a, y → b}
Q(f(x), y) Q(f(a), b) {x → a, y → b}
P(x, a) P(b, y) {x → b, y → a}
Lifting:
Lifting means applying inference rules at the First Order Logic (FOL) level using variables instead of specific constants, making rules generalizable and scalable.
Without Lifting (Propositional) With Lifting (FOL)
If it rains → ground is wet
(Only for this specific case)		 ∀x Rain(x) → Wet(x)
(Works for any x — Paris, London, etc.)
हिंदी
Unification (एकीकरण):
Unification वह प्रक्रिया है जिसमें हम variables को ऐसे values assign करते हैं कि दो logical expressions identical बन जाएं।
उदाहरण: P(x, a) और P(b, y) को same बनाने के लिए x=b और y=a assign करो
Substitution: {x → b, y → a} → दोनों P(b, a) बन जाते हैं

Lifting (उन्नयन):
Lifting का मतलब है inference rules को specific cases की बजाय general variables पर apply करना (FOL level पर)।
बिना Lifting: "अगर बारिश हो तो ज़मीन गीली होगी" (सिर्फ इस एक case के लिए)
Lifting के साथ: ∀x Rain(x) → Wet(x) (किसी भी x के लिए — Paris, Mumbai, etc.)

अंतर: Lifting rules को reusable और scalable बनाती है।

Types of Reasoning in AI
AI में तर्क के प्रकार (Types of Reasoning)
⚡ High ▼
English
Type How it works Example
Deductive General rule → specific conclusion. Always valid if premises are true. All cats have tails. Whiskers is a cat. → Whiskers has a tail.
Inductive Specific observations → general conclusion. Not always certain. I observed 100 crows, all black → All crows are black.
Abductive Best explanation for given evidence. Wet umbrella + dark clouds → "It likely rained".
Probabilistic Uses probability to handle uncertainty. Patient has 70% chance of flu based on symptoms.
Rule-based Predefined IF-THEN rules. IF raining THEN take umbrella.
Case-based Solve new problems using past similar cases. Customer A liked movie X, Customer B similar → recommend X to B.
हिंदी
AI में reasoning के मुख्य प्रकार:

1. Deductive (निगमनात्मक): सामान्य नियम → विशिष्ट निष्कर्ष। यह हमेशा valid होता है।
उदाहरण: सभी पक्षी उड़ते हैं + Robin एक पक्षी है → Robin उड़ती है।

2. Inductive (आगमनात्मक): विशिष्ट observations → सामान्य conclusion। हमेशा सही नहीं होता।
उदाहरण: 10 सेब देखे, सब लाल थे → सभी सेब लाल होते हैं।

3. Abductive (अपकर्षी): दिए गए evidence की best explanation देना।
उदाहरण: गीली छतरी और काले बादल → "शायद बारिश हुई थी"।

4. Probabilistic (संभाव्य): Probability से uncertainty handle करना।
5. Rule-based: IF-THEN rules से decision लेना।
6. Case-based: पुराने similar cases से नई problem solve करना।

What is Planning? (Components & Need)
Planning क्या है? (घटक और आवश्यकता)
🔥 Very High ▼
English
Planning in AI is the process of generating a sequence of actions that transforms an initial state into a desired goal state. It is essential when a problem requires multiple steps and each step depends on previous ones.
Why is Planning needed?
▸Simple reflex agents can only react to current perceptions. Planning enables agents to think ahead.
▸Real-world problems require sequencing actions, handling dependencies, and managing resources.
▸Example: A robot can't just "pick up the package" — it must first move to the right room.
Key Components of Planning:
▸Initial State: The description of the world before any action is taken. E.g., Robot is in Room1, Package is in Room2.
▸Goal State: The desired outcome. E.g., Package is in Room1.
▸Actions (Operators): What the agent can do. Each action has Preconditions (what must be true before) and Effects (what changes after).
▸Plan: The sequence of actions to go from initial state to goal state.
Example — Robot Delivery
Initial: Robot at Room1, Package at Room2
Goal: Package at Room1
Plan: Move(Room1→Hallway), Move(Hallway→Room2), Pick(Package), Move(Room2→Hallway), Move(Hallway→Room1), Drop(Package)
Each step is an action with preconditions (where robot must be) and effects (where robot/package ends up).
Types of Planning:
▸Forward Planning (STRIPS-based) — search forward from initial state
▸Backward Planning — search backward from goal
▸Partial Order Planning — steps not strictly ordered; handles concurrent actions
▸Hierarchical Planning — breaks complex tasks into subtasks
हिंदी
AI में Planning वह process है जिसमें एक agent initial state से goal state तक पहुंचने के लिए actions का sequence तैयार करता है।
Planning की ज़रूरत क्यों?
• Simple agents सिर्फ current situation पर react करते हैं। Planning agent को "सोचने" की क्षमता देती है।
• Real-world tasks में multiple steps होते हैं जो एक-दूसरे पर depend करते हैं।

Planning के मुख्य घटक:
1. Initial State: शुरुआत की स्थिति — जैसे "Robot Room1 में है, Package Room2 में है"
2. Goal State: वांछित स्थिति — जैसे "Package Room1 में हो"
3. Actions/Operators: हर action के Preconditions (पूर्व शर्तें) और Effects (परिणाम) होते हैं
4. Plan: Initial से Goal तक पहुंचने का steps का क्रम

Types: Forward, Backward, Partial Order, Hierarchical Planning
2
Forward Planning & STRIPS Notation
आगे की योजना और STRIPS संकेतन
🔥 Very High ▼
English
STRIPS (Stanford Research Institute Problem Solver) is a classical planning language. Each action is described by: Preconditions (what must be true), Add-list (what becomes true), and Delete-list (what becomes false) after the action.
STRIPS Action Format:
Action Name
Preconditions: [conditions that must hold]
Effects (Add): [facts that become true]
Effects (Delete): [facts that become false]
Solved TA-2 Q3a — Package Swap Problem:
Problem Setup
Locations: Room1, Hallway, Room2 (Room1 ↔ Hallway ↔ Room2)
Initial: At(Robot, Room1), At(Pkg1, Room1), At(Pkg2, Room2), EmptyHanded(Robot)
Goal: At(Pkg1, Room2), At(Pkg2, Room1)
STRIPS Action Definitions:
Move(R1, R2)
Preconditions: At(Robot, R1), Adjacent(R1, R2)
Add: At(Robot, R2)
Delete: At(Robot, R1)
Pick(P, R)
Preconditions: At(Robot, R), At(P, R), EmptyHanded(Robot)
Add: Holding(Robot, P)
Delete: At(P, R), EmptyHanded(Robot)
Drop(P, R)
Preconditions: At(Robot, R), Holding(Robot, P)
Add: At(P, R), EmptyHanded(Robot)
Delete: Holding(Robot, P)
Step-by-Step Execution Plan:
1
Pick(Pkg1, Room1) — Robot picks Pkg1 from Room1
2
Move(Room1, Hallway) — Robot moves to Hallway
3
Move(Hallway, Room2) — Robot moves to Room2
4
Drop(Pkg1, Room2) — Robot drops Pkg1 in Room2 ✓ [Goal 1 done]
5
Pick(Pkg2, Room2) — Robot picks Pkg2 from Room2
6
Move(Room2, Hallway) → Move(Hallway, Room1) — Robot returns
7
Drop(Pkg2, Room1) — Robot drops Pkg2 in Room1 ✓ [Goal 2 done]
Limitation of STRIPS: Cannot handle concurrent actions, resources, or time constraints easily.
हिंदी
STRIPS एक classical planning method है। हर action को तीन भागों में describe किया जाता है:
• Preconditions: क्या सत्य होना चाहिए पहले
• Add-list: action के बाद क्या नया सत्य होगा
• Delete-list: action के बाद क्या असत्य होगा
Package Swap Problem का plan:
1. Pkg1 उठाओ (Room1 से)
2. Hallway में जाओ
3. Room2 में जाओ
4. Pkg1 रखो (Room2 में) ✓
5. Pkg2 उठाओ (Room2 से)
6. वापस Room1 में जाओ
7. Pkg2 रखो (Room1 में) ✓

STRIPS की सीमा: यह concurrent actions, time, और resources को handle नहीं कर सकता।

CSP as a Search Problem
बाधा संतुष्टि समस्या एक खोज समस्या के रूप में
🔥 Very High ▼
English
A Constraint Satisfaction Problem (CSP) is a problem where we must assign values to variables such that all constraints are satisfied. It is naturally framed as a search problem.
3 Elements of a CSP:
▸Variables: X = {X₁, X₂, ..., Xₙ} — the unknowns to be assigned
▸Domains: D = {D₁, D₂, ..., Dₙ} — possible values each variable can take
▸Constraints: C = rules restricting which value combinations are allowed
CSP as Search Problem:
▸State: A partial or complete assignment of values to variables
▸Initial State: Empty assignment (no variables assigned)
▸Goal State: Complete assignment that satisfies all constraints
▸Successor Function: Assign a value to one unassigned variable
Classic Example — Map Coloring (Australia)
Variables: WA, NT, Q, NSW, V, SA, T (Australian states)
Domains: {Red, Green, Blue} for each state
Constraint: No two adjacent states can have the same color
Solution: WA=Red, NT=Green, Q=Red, NSW=Green, V=Red, SA=Blue, T=Green

Search explores assignments, backtracks when a constraint is violated.
Another Example — Sudoku
Variables: 81 cells
Domains: {1,2,3,4,5,6,7,8,9}
Constraints: Each row, column, and 3×3 box must have all 9 digits exactly once
Types of Constraints:
▸Unary: Single variable — e.g., SA ≠ green
▸Binary: Two variables — e.g., SA ≠ WA
▸Higher-order: 3+ variables — e.g., cryptarithmetic constraints
हिंदी
CSP (Constraint Satisfaction Problem) वह problem है जिसमें variables को values assign करनी होती हैं जो सभी constraints को satisfy करें।
CSP के 3 मुख्य भाग:
1. Variables (चर): जो unknowns हैं — जैसे Australian states
2. Domains (क्षेत्र): हर variable के possible values — जैसे {Red, Green, Blue}
3. Constraints (बाधाएँ): नियम — जैसे "adjacent states का color same नहीं होना चाहिए"

Search Problem के रूप में:
• Start: कोई assignment नहीं
• Goal: सभी variables को ऐसे assign करो कि सभी constraints satisfy हों
• Search: एक-एक variable को assign करते जाओ, violation पर backtrack करो

उदाहरण: Map Coloring (Australia), Sudoku, Scheduling
4
Backtracking Search for CSP
CSP के लिए Backtracking Search
🔥 Very High ▼
English
Backtracking Search is a depth-first search algorithm that assigns values to variables one at a time. If an assignment violates a constraint, it backtracks and tries the next value.
Algorithm Steps:
1
Initialize: Start with empty assignment.
2
Select: Pick an unassigned variable (e.g., using MRV heuristic).
3
Assign: Try a value from its domain.
4
Check: If the assignment is consistent (no constraint violated), continue recursively.
5
Backtrack: If inconsistent or dead end → undo the assignment, try next value.
6
Terminate: All variables assigned → solution found. No more values → failure.
Example — Map Coloring (WA, NT, SA)
WA = Red → NT (adjacent to WA) can't be Red → NT = Green
SA (adjacent to WA and NT) can't be Red or Green → SA = Blue ✓
Continue assigning Q, NSW, V, T using same logic.
If at any point no valid color → backtrack and try a different color for previous state.
Optimization Techniques:
▸MRV (Minimum Remaining Values): Choose the variable with fewest legal values first — catches conflicts early
▸Forward Checking: When a variable is assigned, eliminate inconsistent values from neighbors' domains — prevents future failures
▸LCV (Least Constraining Value): Choose the value that rules out fewest choices for neighbors
Advantages: Simple to implement, works well with heuristics.
Disadvantages: Slow for large problems without optimization, may explore many invalid paths.
हिंदी
Backtracking Search एक depth-first search है जिसमें variables को एक-एक करके values assign की जाती हैं। अगर कोई constraint violate हो जाए, तो पिछले step पर वापस जाते हैं (backtrack) और दूसरी value try करते हैं।
Algorithm steps:
1. Empty assignment से शुरू करो
2. एक unassigned variable चुनो
3. उसे domain से कोई value assign करो
4. Constraint check करो — अगर violation नहीं तो आगे बढ़ो
5. अगर violation हो → backtrack करो और दूसरी value try करो
6. जब सभी variables assign हो जाएं → solution मिल गया ✓

Optimization:
• MRV: सबसे कम options वाला variable पहले select करो
• Forward Checking: Assignment के बाद neighbors के domain से invalid values हटाओ
• LCV: वह value चुनो जो neighbors की domain को कम restrict करे

Hierarchical Planning
श्रेणीबद्ध योजना (Hierarchical Planning)
⚡ High ▼
English
Hierarchical Planning (also called HTN — Hierarchical Task Network Planning) breaks complex tasks into smaller, manageable subtasks at multiple levels of abstraction.
How it works:
1
Start with a high-level abstract goal (e.g., "Travel from City A to City B")
2
Decompose into sub-tasks (e.g., "Book ticket", "Pack bags", "Go to airport")
3
Further decompose sub-tasks into primitive actions (e.g., "Buy ticket online" → open website, enter details, pay)
4
Execute primitive actions in order
Example — Make Coffee
Level 1 Goal: Make Coffee
Level 2 Sub-tasks: Boil water, Add coffee powder, Pour in cup
Level 3 Primitive actions for "Boil water": Fill kettle → Plug in → Switch on → Wait until boiled

Each level decomposes until we reach basic executable actions.
Advantage: Handles complex real-world tasks by dividing them into manageable pieces. Easier to reason about.
Challenge: Designing the right decomposition hierarchy is difficult. Multi-agent planning adds coordination challenges.
हिंदी
Hierarchical Planning में एक बड़े goal को छोटे-छोटे sub-tasks में तोड़ा जाता है, और फिर हर sub-task को और छोटे steps में।
उदाहरण — "Coffee बनाओ":
Level 1: Coffee बनाओ
Level 2: पानी उबालो, Coffee powder डालो, Cup में डालो
Level 3: "पानी उबालो" के लिए → Kettle भरो → Plug in करो → On करो → Wait करो

फायदे:
• Complex tasks को manage करना आसान
• Higher level पर reasoning करना सरल

चुनौती:
• Decomposition hierarchy सही तरीके से design करना मुश्किल है
• Multi-agent scenario में coordination की ज़रूरत होती है

Partial Order Planning
आंशिक क्रम योजना (Partial Order Planning)
⚡ High ▼
English
Partial Order Planning (POP) does not commit to a total ordering of all steps. It only specifies an order between steps when necessary (when one step must come before another to avoid conflicts). Steps that don't interfere can remain unordered.
Key Idea: "Least commitment" — only order steps when required.

POP Plan consists of:
▸Steps: The actions to be taken
▸Ordering constraints: Step A must come before Step B (A < B)
▸Causal links: Step A produces condition C needed by Step B
▸Open conditions: Preconditions not yet satisfied
Example — Baking a Cake (TA-2 Q7B pattern)
Steps: Preheat oven, Mix ingredients, Pour batter, Bake

Constraints:
• Preheat must happen BEFORE Bake (oven needs to be hot)
• Mix must happen BEFORE Pour (need mixed batter to pour)
• Pour must happen BEFORE Bake (batter must be in pan)

But: Preheat and Mix can happen in ANY order (or simultaneously) — no conflict between them.
POP allows this flexibility: Preheat ∥ Mix → Pour → Bake
Advantage over total-order planning: More flexible, can exploit parallelism, generates a family of plans.
हिंदी
Partial Order Planning में steps के बीच तभी order define किया जाता है जब ज़रूरी हो। जिन steps के बीच conflict नहीं, उन्हें unordered रखा जाता है।
उदाहरण — Cake बनाना:
Steps: Oven Preheat, Mix ingredients, Pour, Bake

ज़रूरी order:
• Mix → Pour (पहले mix फिर pour)
• Pour → Bake (पहले pour फिर bake)
• Preheat → Bake (preheat हो तभी bake होगा)

लेकिन Preheat और Mix को एक साथ किया जा सकता है — कोई conflict नहीं!

फायदा: Steps को parallel में execute किया जा सकता है → time efficient।

Goal Stack Planning & Blocks World
Goal Stack Planning और Blocks World
⚡ High ▼
English
Goal Stack Planning uses a stack data structure to manage goals and sub-goals. The main goal is pushed onto the stack, decomposed into sub-goals, and actions are executed until all goals are achieved.
Blocks World: A classic planning domain where blocks on a table must be rearranged to form a goal configuration.
Classic Blocks World Example
Initial: A on B, B on table, C on table, A is clear, C is clear
Goal: A on C, C on B, B on table

Stack initially: [Goal: {on(A,C), on(C,B), on(B,table)}]

Plan execution:
1. Unstack(A,B) — pick A off B [Precondition: clear(A), on(A,B)]
2. PutDown(A) — place A on table
3. PickUp(C) — pick C from table
4. Stack(C,B) — place C on B ✓
5. PickUp(A) — pick A from table
6. Stack(A,C) — place A on C ✓

Final: A on C on B on Table ✓ [Goal achieved]
Key Predicates: on(X,Y), clear(X), holding(X), on_table(X)
Key Actions: PickUp, PutDown, Stack, Unstack
हिंदी
Goal Stack Planning में एक stack का उपयोग होता है जिसमें goals और sub-goals store किए जाते हैं। Main goal को stack पर push किया जाता है, sub-goals में तोड़ा जाता है, और actions execute होती हैं।
Blocks World: Table पर रखे blocks को goal configuration में arrange करना।

उदाहरण:
Initial: A, B पर है। C table पर है।
Goal: A, C पर हो। C, B पर हो।

Plan:
1. A को B से उठाओ (Unstack A from B)
2. A को table पर रखो (PutDown A)
3. C उठाओ (PickUp C)
4. C को B पर रखो (Stack C on B) ✓
5. A उठाओ (PickUp A)
6. A को C पर रखो (Stack A on C) ✓

Result: A → C → B → Table ✓
